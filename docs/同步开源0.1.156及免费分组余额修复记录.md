## 免费分组余额修复

问题位置在 `BillingCacheService.CheckBillingEligibility`：中间件已对免费分组跳过余额检查，但网关处理器的二次计费资格检查仍按普通余额分组拒绝余额为零的用户。

修复规则：

- 仅当 `group.IsFree == true` 时跳过余额资格检查。
- 免费分组仍然受每日免费额度、用户平台配额、API Key 限流和 RPM 限制约束。
- 普通余额分组和订阅分组的既有余额/订阅校验不变。
- 新增单元测试验证：免费分组余额为零可调用；普通分组余额为零仍返回余额不足。
