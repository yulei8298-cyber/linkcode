## 二、实现内容

### 1. 分组配置

分组新增以下字段，并同步到 Ent、管理 API、DTO、前端类型和管理页面：

- `is_hidden`：对普通用户隐藏分组和该分组下的密钥。
- `is_free`：启用按用户、分组、自然日核算的免费计费。
- `daily_free_limit_usd`：每日免费额度，必须大于 0。
- `chat_station_only`：只允许携带正确对话站共享密钥的请求调用。

免费分组只能使用 `standard` 模式；关闭免费开关时会清空每日免费额度。管理员仍可查看和管理隐藏密钥，普通用户不能创建、改绑、查看、修改、删除隐藏密钥，也不能通过用量接口查询隐藏密钥。

### 2. 数据迁移

新增迁移 `backend/migrations/177_chat_station_free_groups.sql`：

- 增加四个分组字段和数据库约束。
- 新建 `daily_free_usages` 每日免费用量账本，唯一键为用户、分组、日期。
- 新建对话站自动密钥唯一索引，避免首聊并发创建重复密钥。
- 旧 `OpenAI-chat` 仅在不存在人工/付费订阅时转换为隐藏、免费、仅对话站的标准分组，并沿用原每日额度。
- 如果同组存在人工或付费订阅，保留原分组的订阅模式和授权，只失效备注为 `auto assigned by LobeHub SSO` 的旧自动订阅。
- 如果旧 `OpenAI-chat` 没有有效的正数每日额度，迁移直接失败，避免生成无限免费配置。

### 3. 鉴权与首次无密钥调用

对话站服务端使用以下协议：

| 方向 | Header | 用途 |
| --- | --- | --- |
| Lobe -> 中转站 | `X-LinkCode-Chat-Station-Secret` | 校验可信对话站来源 |
| Lobe -> 中转站 | `X-LinkCode-Chat-Station-User-ID` | 共享密钥校验成功后识别 LinkCode 用户 |
| 中转站 -> Lobe | `X-LinkCode-Chat-Station-API-Key` | 首聊自动创建或复用的密钥 |

无 API Key 时，中转站根据请求路径识别 OpenAI 或 Anthropic，选择满足 `active + standard + is_hidden + is_free + chat_station_only` 的平台分组，按 `sort_order,id` 选择，只复用专用自动密钥，否则创建新密钥并继续同一次请求。

主要错误契约：

- 非对话站调用受限分组：HTTP 403，`当前分组仅允许通过对话站调用`。
- 每日免费额度耗尽：HTTP 429，`当前每日免费额度已用完，请明天再使用`。
- 没有候选免费分组：HTTP 401，引导对话站进入密钥设置。
- 密钥所属分组已删除或不存在：HTTP 401，`API 密钥无效或所属分组不存在`。

### 4. 免费计费

- 鉴权阶段读取当日账本，达到额度后在调度前拒绝。
- 统一计费事务继续写入实际成本、API Key 限额和账号限额，同时累计 `daily_free_usages`。
- 免费请求不扣用户余额、不累计余额平台配额、不发送余额不足通知。
- Anthropic/通用网关和 OpenAI 专用计费路径均显式标记免费计费。
- Simple Mode 仍只保留原有不扣费行为，但会写入每日免费账本，避免额度永不增长。
- 额度预检日期随请求上下文传到异步计费，跨午夜完成的请求不会改记到次日。
- 统一计费仓储不可用时免费请求失败关闭，避免降级成不记账的无限免费。

### 5. Lobe 改造

- Better Auth `accounts` 保存 Lobe 用户与 LinkCode 用户 ID 的绑定。
- 首聊仅在服务端注入共享密钥和用户 ID，不向浏览器下发共享密钥或自动密钥。
- 删除占位 Authorization/API Key 后再请求中转站。
- 读取响应密钥后使用 KeyVault 加密保存，并自动启用对应 Provider。
- 密钥加密或数据库保存失败时，首次请求显式失败，不吞掉错误返回伪成功。
- OpenAI 使用 `/v1`，Anthropic 使用网关根路径。
- 共享密钥只发送到 LinkCode 官方 HTTPS 地址或明确配置的内部地址；公网 HTTP 地址失败关闭。
- 删除旧 `src/server/services/linkcodeSso.ts` 预创建密钥服务。

## 三、兼容处理与审查修复

- 保留普通余额分组、订阅分组、普通密钥的原鉴权和计费逻辑。
- 鉴权缓存版本升级到 16，并写入免费配置和仅对话站配置。
- 修正 Google/Gemini 分组失效错误，使其与主鉴权统一返回 HTTP 401。
- 修正 OpenAI 专用计费路径遗漏 `IsFreeBill` 的问题，避免免费请求继续扣余额。
- 将隐藏过滤限定在用户侧列表；管理员列表、管理员删除用户和分组密钥管理仍包含隐藏密钥。
- 候选组强制必须隐藏，自动密钥查询强制使用专用名称，避免把普通密钥绑定到 Lobe。
- 停用的自动密钥不再占用唯一索引，首聊可重新创建活动密钥。
- 每日免费额度在服务层限制为小于 `10000000000`，与 `DECIMAL(20,10)` 整数位容量一致。
- 已删除分组仍保留在密钥上的 GroupID，用户侧按不可见处理，鉴权按无效密钥处理。
