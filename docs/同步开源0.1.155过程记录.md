## 七、测试阶段发现并修复的问题

### 7.1 Use Key 弹窗 API Base 兼容

上游改动引入了 `apiBase` 使用，但自动合并后该变量在本地组件结构中不存在，会导致前端类型检查失败，并可能改变 OpenAI/OpenCode 的既有地址语义。

处理结果：

- 仅 Grok 配置使用 `${root}/v1`，变量命名为 `grokAPIBase`。
- OpenAI/OpenCode 继续使用本地网关根地址。
- Antigravity Gemini 和 Anthropic 的现有派生规则保持不变。

### 7.2 Repository SQLite 测试与 IP ACL 迁移兼容

上游新增的 API Key repository 测试使用 Ent 临时 SQLite schema，而本地分组 IP ACL 字段由历史 SQL 迁移添加，不在 Ent schema 中。鉴权查询 join 分组时因此缺少 `ip_whitelist`、`ip_blacklist` 列。

处理结果：

- 测试数据库初始化时补建两个 ACL 列，使其与生产迁移后的 schema 一致。
- `hydrateGroupIPACL` 从 PostgreSQL 专用的 `ANY(array)` 改为参数化 `IN ($1, ...)`。
- PostgreSQL 生产查询语义不变，同时使 SQLite repository 测试可执行。
