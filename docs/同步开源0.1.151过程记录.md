## 三、同步方式

本地二开历史与开源历史不能直接使用普通 merge 得到合适的共同祖先，因此继续使用上一轮同步后的开源标签作为人为共同基线，执行三方合并：

```bash
git merge-recursive v0.1.146 -- HEAD v0.1.151
```

三方含义：

- 基线：`v0.1.146`
- 本地版本：当前二开 `HEAD`
- 上游版本：`v0.1.151`

合并阶段产生 16 个文件级冲突。处理原则为：先确认本地代码解决的业务问题，再将它迁移到上游的新文件结构中；只有确认上游实现已完整覆盖本地能力时，才采用上游版本。

最终相对同步前 `HEAD` 的代码规模约为：468 个文件变更、121400 行新增、58755 行删除。大量增删来自上游对后端大文件和前端语言包的拆分，不代表等量业务逻辑重写。

## 四、上游主要同步内容

- 批量图片任务：新增任务、条目、事件数据模型，队列、Worker、结算、下载、清理、前后端页面和相关迁移。
- OpenAI/Codex：新增 Codex 模型清单接口、GPT-5.6 兼容与计费、Fast/Flex 策略、compact SSE/JSON 桥接、WebSocket 兼容和身份标识修复。
- 网关：增强 `/v1/messages` 回退、Responses 错误透传、非法 JSON 处理、web search 块过滤和请求解析日志。
- Grok：支持 Grok 4.5，完善图片/视频路由、按秒计费、分辨率和推理强度兼容。
- 计费与订阅：冻结余额、订阅窗口同步维护、支付恢复和并发计费加固。
- 管理后台：用户角色编辑、用户 Token 排行、用量页与延迟健康展示、版本在线回退入口。
- 工程结构：拆分 `gateway_service.go`、`openai_gateway_service.go`、`setting_service.go`、`admin_service.go`、`setting_handler.go`、`usage_log_repo.go`、`openai_ws_forwarder.go` 等大文件。
- 前端国际化：原 `zh.ts`、`en.ts` 拆分为按业务域组织的语言包，并新增键冲突测试。
- 安全与依赖：升级 Go 工具链引用到 1.26.5，修复 AWS SDK 漏洞，统一站点 Logo 和文档 URL 清洗。

### 5.2 Handler 与依赖注入

- 保留本地 `PublicPricingHandler` 和 `LobeHubSSOHandler`。
- 合入上游 `BatchImageHandler` 及其服务、仓库、Worker、清理任务。
- 手工兼容 `handler.go`、`wire.go`、`wire_gen.go`，保证三类 Handler 同时注入。

### 5.3 API 网关路由

- 保留本地根路径兼容路由，例如 `/messages`、`/responses`、`/chat/completions`。
- 保留本地 `/v1/v1/*`、`/antigravity/v1/v1/*` 旧客户端兼容路由。
- 合入批量图片、Grok 视频和 Codex 模型清单路由。
- `/models` 根据请求兼容两种返回：OpenAI 平台且带 `client_version` 时返回 Codex manifest，其他请求继续返回原 OpenAI 风格模型列表。

### 5.4 API Key 鉴权与调用兼容

- Gemini 鉴权同时保留分组 IP ACL、Key IP ACL、专属分组授权和本地对话站密钥限制。
- 合入上游 IP 拒绝信息、Ops 业务限流标记、过期/额度耗尽状态校验。
- 订阅窗口需要维护时先同步刷新再校验，避免旧窗口数据导致错误拒绝。
- 余额鉴权继续保持本地历史语义：仅 `balance <= 0` 时拒绝，不把 `MinimumBalanceReserve` 变成所有 API 的硬门槛。
- API Key 鉴权缓存快照升级为 `v14`，同时覆盖分组 IP ACL、高峰倍率和视频定价字段。

### 5.5 分组与系统设置

上游把原有大文件拆分后，本地二开字段迁移到新的职责文件中：

- `admin_group.go`：保留分组 IP 白名单、黑名单的 DTO、校验和写入。
- `setting_public.go`：保留 `linkcode` 默认品牌、定价展示、教程和对话站公开配置。
- `setting_update.go`、`setting_parse.go`、`setting_features.go`：保留二开设置键的读取、解析、默认值和更新。
- `setting_handler_update.go`、`setting_handler_audit.go`：保留后台设置接口和审计字段。

采用上游拆分后的文件结构，不恢复旧版大文件，降低以后再次同步的冲突范围。
