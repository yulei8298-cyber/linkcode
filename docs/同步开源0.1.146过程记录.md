# 同步开源 0.1.146 过程记录

同步日期：2026-07-07

## 背景

当前中转站二开版本号为 `0.1.138`，开源仓库 `Wei-Shaw/sub2api` 最新标签为 `v0.1.146`。本次任务是在不提交代码的前提下，将开源 `v0.1.146` 的最新代码同步到当前二开仓库，并优先保护现有二开能力。

## 同步方式

1. 执行 `git fetch upstream --tags` 获取开源最新分支和标签。
2. 确认本地工作树同步前是干净状态。
3. 创建备份分支 `backup/pre-sync-v0.1.146-20260707` 指向同步前的本地 `HEAD`。
4. 由于本地二开历史与开源标签没有 Git 共同祖先，采用人为三方合并方式：

   ```bash
   git merge-recursive v0.1.138 -- HEAD v0.1.146
   ```

   其中 `v0.1.138` 作为共同基线，当前 `HEAD` 作为本地二开版本，`v0.1.146` 作为开源目标版本。

## 主要同步内容

- 版本号更新到 `backend/cmd/server/VERSION = 0.1.146`。
- 同步开源 `0.1.139` 至 `0.1.146` 的后端、前端、部署和资源更新。
- 纳入开源新增能力，包括但不限于：
  - OpenAI Responses compact 端点归一化修复。
  - OpenAI/Grok 图像与视频网关能力。
  - OpenAI `count_tokens` 兼容桥。
  - Grok OAuth、Grok quota、Grok media 相关服务与前端入口。
  - OpenAI 高级调度器、API Key 并发统计、账号请求头覆写。

## 冲突处理原则

- 本地二开功能优先保留。
- 开源安全修复、兼容性修复和新增字段尽量合入。
- 当本地与开源发生语义冲突时，优先采用兼容合并；无法兼容时保留本地逻辑。
